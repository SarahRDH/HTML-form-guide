 Gives the user an opportunity to provide information which is determined by the parameters that follow the <input> tag.
The HTML <input> element is used to render a variety of input fields on a webpage including text fields, checkboxes, buttons, etc.
<input> element have a type attribute that determines how it gets rendered to a page.
By including "submit", a submit button will be rendered on the webpage.
The data will not be sent correctly unless the Key/Value pair is set up properly.
The KEY is the name="xyz" of the <input>. The VALUE is whatever the user entered for that question on the form.
This is how the server connects the user's answers to the appropriate questions.

 Use the "post" method for large amounts of data that may contain sensitive information.
Use the "get" method for limited quantities of data that are not secure.
Get is the default setting in a form and the data will be visible in the URL.
Use GET for retrieving / viewing information (such as entering a search request into a text box in order to view static posts from other users) and POST for creating / editing information (such as having a user complete a questionnaire with unique answers for each user).
The method is how the servers communicate with each other.
The action is the address of the server that will receive the user's data.
The target="value" determines where the form results or submission success page will land.		 <form method="post" action="http://exampleserver1">

Key Points

  • Labels and IDs must be the same for ADA screenreaders.
  • Names and Values must be associated with eachother for each form item completed by a user or the input will not be connected to anything on submission.
  • "Submit" will not work unless "action" is specified.
